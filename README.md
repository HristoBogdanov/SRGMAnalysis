# Тестови данни

Разглежданата софтуерна система е dotnet/runtime — официалното хранилище на .NET Runtime, поддържано от Microsoft и общността с отворен код, което включва CoreCLR, CoreFX, Mono runtime и стандартните библиотеки на платформата. Данните са извлечени чрез GitHub REST API с помощта на C# приложение, което агрегира броя на докладваните issues по дни. Те представляват кумулативен брой открити дефекти за период от 48 дни, като за това време са регистрирани общо 1900 issues, при среден темп от около 39–40 нови issues на ден. Данните са извлечени от последните докладвани issues в хранилището, което съответства на фазата на поддръжка и тестване след пускане (post-release / operational testing) — дефектите се откриват и докладват от реални потребители и разработчици в работна среда, след като дадена версия на .NET вече е публично достъпна. Това е стандартна практика при проекти с отворен код, при които крайните потребители са едновременно и тестери.

# Rayleigh Модел

Rayleigh моделът е модел за анализ на надеждността на софтуер, базиран на нехомогенен Поасонов процес (NHPP), при който интензивността на откриване на дефекти следва форма на Rayleigh разпределение. Моделът е предназначен за системи с голям мащаб и сложност, при които броят на откритите дефекти първоначално нараства, достига пик и след това намалява — типична S-образна крива. Особено подходящ е за оперативни системи в активна фаза на поддръжка, каквато е dotnet/runtime, където потокът от нови issues нараства постепенно преди да се стабилизира.

### Средната стойностна функция на модела е:

`m(t) = a · [1 − e^(−bt²)]`

- a - общият очакван брой дефекти в системата (асимптота на кумулативната крива)
- b - параметър на мащаба, който контролира скоростта на нарастване и положението на пика на интензивност на откриване на дефекти
- t - времето на тестване/наблюдение
